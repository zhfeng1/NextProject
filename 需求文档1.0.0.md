# NextProject 需求文档 v1.0.0

## 项目概述

NextProject 是一个基于 AI 的智能网站开发平台，支持通过多种 AI 代理（Codex、Claude Code、Gemini CLI）自动化生成、调整和部署网站。

## 系统架构

### 1. 服务组件

#### 1.1 主服务 (main-service)
- 端口：18080
- 技术栈：Python FastAPI + SQLite
- 功能：
  - 全局配置管理
  - 子网站生命周期管理
  - 任务调度和执行
  - AI Provider 集成
  - 部署流程控制

#### 1.2 Codex MCP 服务 (codex-mcp)
- 端口：8090（内部）
- 功能：
  - Codex OAuth 认证管理
  - MCP Server 自动启动和监控
  - 认证状态查询

#### 1.3 子网站系统
- 动态端口范围：19100-19999
- 默认技术栈：Python FastAPI（后端）+ Vue 3（前端）
- 隔离运行，通过主服务反向代理访问

### 2. 数据库设计

#### 2.1 核心表结构

**app_config**（全局配置）
- LLM 配置：模式、API 密钥、模型选择
- Codex OAuth 配置：客户端 ID、密钥、访问令牌
- MCP URL 配置

**sub_sites**（子网站）
- site_id（主键）
- name（网站名称）
- status（运行状态）
- port（内部端口）
- 时间戳

**site_deploy_config**（部署配置）
- target_type：local / apollo
- Apollo 配置：系统 API、Harbor 配置、登录凭据
- 模块名称和路径配置

**site_provider_config**（Provider 配置）
- codex_cmd / claude_cmd / gemini_cmd
- 对应的认证命令

**agent_tasks**（任务记录）
- id（UUID）
- site_id、provider、task_type
- status：queued / running / success / failed / canceled
- payload_json、result_json
- 时间戳

**agent_task_logs**（任务日志）
- task_id、timestamp、level、line

## 功能需求

### 3. 核心功能

#### 3.1 网站管理

**创建网站**
- 输入：site_id（可选）、name（可选）
- 自动分配端口
- 初始化目录结构：backend/、frontend/
- 生成默认模板代码
- 自动启动子网站进程

**启动/停止网站**
- 启动：通过 uvicorn 启动子网站 FastAPI 应用
- 停止：优雅终止进程（SIGTERM → SIGKILL）
- 进程管理：线程锁保护，防止竞态

**删除网站**
- 停止进程
- 删除数据库记录（级联删除相关任务和日志）
- 删除文件系统目录

**预览访问**
- 统一入口：http://localhost:18080/sites/{site_id}
- 主服务反向代理到子网站内部端口
- 自动处理路径前缀

#### 3.2 AI 开发任务

**支持的 Provider**
- Codex（默认）
- Claude Code
- Gemini CLI

**开发任务流程**
1. 用户提交 prompt / instruction / command
2. 创建任务记录（状态：queued）
3. 任务入队，worker 异步处理
4. 执行 CLI 命令（cwd=网站根目录）
5. 实时日志流式写入数据库
6. 更新任务状态和结果

**任务类型：develop_code**
- 参数：prompt（需求描述）或 command（自定义命令）
- Provider 命令可按站点配置覆盖
- 支持取消（通过 cancel flag）

#### 3.3 测试功能

**Playwright 本地冒烟测试**
- 任务类型：test_local_playwright
- 通过 Node.js 脚本调用 Playwright
- 测试入口：可配置 base_url（默认 http://127.0.0.1:8080）
- 容器内自动检测和 URL 调整
- 截图和报告保存到 artifacts 目录

#### 3.4 部署功能

**本地部署（deploy_local）**
- 重启子网站服务
- 返回预览 URL

**Apollo 部署（deploy_apollo）**
1. 动态生成 Dockerfile.deploy
2. Docker 构建镜像（时间戳标签）
3. 推送到 Harbor 私有仓库
4. 登录 Apollo 系统
5. 调用部署接口（deployByImage）
6. 清理本地镜像

**配置要求**
- system_api_base、system_id、app_id
- harbor_domain、harbor_namespace
- 登录凭据：tel、password、random

#### 3.5 LLM 集成

**支持模式**
- chat_completions（OpenAI 标准）
- responses（Anthropic 风格）

**应用场景**
- 网站规划：需求 → JSON（title、summary）
- 网站编辑：指令 → JSON（title、note、requirement_append）

**配置**
- llm_base_url（默认：https://api.openai.com/v1）
- llm_api_key
- llm_model（默认：gpt-4.1-mini）

**对话日志**
- 可选开启：LLM_DIALOG_LOG_ENABLED
- 字段截断：LLM_DIALOG_LOG_MAX_CHARS

#### 3.6 Provider 认证

**多 Provider 认证流程**
- Codex：codex login --device-auth
- Claude Code：claude login
- Gemini CLI：gemini auth login

**认证管理**
- 启动认证流程：异步进程，实时日志
- 查询状态：进程运行状态 + auth.json 文件检测
- 取消流程：SIGTERM 终止进程

**状态检测**
- authenticated：auth 文件存在且非空
- running：进程 poll() is None
- recent_logs：最近 50 条日志

### 4. 任务系统

#### 4.1 任务队列

**架构**
- queue.Queue：线程安全队列
- Worker 数量：可配置（默认 2）
- 后台线程池：daemon 模式

**执行流程**
1. 任务创建 → 状态：queued
2. enqueue_task → 进入队列
3. worker 取出任务 → 状态：running
4. 执行具体逻辑（develop / test / deploy）
5. 更新状态：success / failed / canceled

**取消机制**
- 全局 cancel flag 集合（线程锁保护）
- 轮询检测：run_shell_command_with_logs 中每读一行检查
- 立即终止：subprocess.terminate()

#### 4.2 日志管理

**增量读取**
- GET /api/tasks/{task_id}/logs?after_id=N
- 返回：logs、next_after_id
- 前端轮询实现实时日志流

**日志级别**
- INFO：常规输出
- WARN：取消、警告
- ERROR：异常、失败

#### 4.3 任务历史

**站点任务列表**
- GET /api/sites/{site_id}/tasks
- 按 created_at DESC 排序
- 分页限制（默认 30）

## 非功能需求

### 5. 性能要求

- 任务并发：2 workers（可配置）
- 命令超时：1800 秒（30 分钟）
- HTTP 超时：20-30 秒
- 日志缓冲：200 行（tail）

### 6. 安全要求

- API 密钥存储：SQLite（明文，需加密改进）
- Docker socket 挂载：仅主服务容器
- 容器隔离：子网站进程在主容器内运行
- 网络隔离：子网站端口不对外暴露

### 7. 可靠性

- 服务重启恢复：
  - 标记所有 queued/running 任务为 failed
  - 自动重启 status=running 的子网站
- 进程监控：
  - site_processes 字典管理进程生命周期
  - poll() 检测进程退出，自动清理
- 优雅关闭：
  - TASK_STOP_EVENT 通知 workers 停止
  - 终止所有子网站进程

### 8. 可扩展性

- 端口池：19100-19999（900 个子网站）
- Provider 扩展：SUPPORTED_PROVIDERS 字典
- 任务类型扩展：SUPPORTED_TASK_TYPES 集合
- LLM 模式切换：responses / chat_completions

- ./data:/shared（数据持久化）

## 已知限制

### 13. 当前限制

1. 子网站在主容器内运行（非独立容器）
2. API 密钥明文存储
3. 无用户认证和权限控制
4. 无 HTTPS 支持
5. 日志无持久化清理策略
6. 任务队列无持久化（重启丢失）
7. 子网站端口池固定（无动态扩展）
8. Playwright 测试仅支持本地运行
