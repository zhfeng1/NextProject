### 1. SECRET_KEY 硬编码导致 JWT 安全风险

**严重性**: 🔴 CRITICAL
**影响范围**: 所有用户认证和授权功能
**风险**: 攻击者可以伪造任意用户 Token，完全绕过认证

#### 当前问题

**文件**: `backend/core/config.py`
```python
secret_key: str = Field(default="change-me-in-production", alias="SECRET_KEY")
```

**文件**: `.env.example`
```bash
SECRET_KEY=change-me-in-production
```

**文件**: `docker-compose.yml`
```yaml
environment:
  - SECRET_KEY=${SECRET_KEY:-change-me-in-production}
```

#### 修复方案

**步骤 1**: 更新 `backend/core/config.py`
```python
import secrets
from pydantic import Field, field_validator

class Settings(BaseSettings):
    secret_key: str = Field(alias="SECRET_KEY")

    @field_validator("secret_key")
    @classmethod
    def validate_secret_key(cls, v: str) -> str:
        if v == "change-me-in-production":
            raise ValueError(
                "SECRET_KEY must be changed from default value. "
                "Generate a secure key with: python -c 'import secrets; print(secrets.token_urlsafe(32))'"
            )
        if len(v) < 32:
            raise ValueError("SECRET_KEY must be at least 32 characters long")
        return v
```

**步骤 2**: 生成生产密钥
```bash
# 生成 256-bit 安全密钥
python -c 'import secrets; print(secrets.token_urlsafe(32))'
```

**步骤 3**: 更新 `.env` 文件（不要提交到 Git）
```bash

```

**步骤 4**: 更新 `.env.example`

```python
from minio import Minio
from minio.error import S3Error
from pathlib import Path
from backend.core.config import settings
import logging

logger = logging.getLogger(__name__)

class MinioClient:
    def __init__(self):
        self.client = Minio(
            endpoint=settings.minio_endpoint,  # "minio:9000"
            access_key=settings.minio_access_key,
            secret_key=settings.minio_secret_key,
            secure=settings.minio_secure  # False for HTTP
        )

    def _ensure_bucket(self, bucket: str):
        """确保 Bucket 存在"""
        try:
            if not self.client.bucket_exists(bucket):
                self.client.make_bucket(bucket)
                logger.info(f"Created bucket: {bucket}")
        except S3Error as e:
            logger.error(f"Bucket check failed: {e}")
            raise

    async def upload_file(self, file_path: str | Path, bucket: str, object_name: str) -> str:
        """上传文件到 MinIO"""
        self._ensure_bucket(bucket)

        try:
            self.client.fput_object(bucket, object_name, str(file_path))
            logger.info(f"Uploaded {file_path} to {bucket}/{object_name}")
            return f"{bucket}/{object_name}"
        except S3Error as e:
            logger.error(f"Upload failed: {e}")
            raise Exception(f"MinIO upload failed: {e}")

    async def download_file(self, bucket: str, object_name: str, file_path: str | Path) -> Path:
        """从 MinIO 下载文件"""
        try:
            self.client.fget_object(bucket, object_name, str(file_path))
            logger.info(f"Downloaded {bucket}/{object_name} to {file_path}")
            return Path(file_path)
        except S3Error as e:
            logger.error(f"Download failed: {e}")
            raise Exception(f"MinIO download failed: {e}")

    async def delete_file(self, bucket: str, object_name: str) -> bool:
        """删除文件"""
        try:
            self.client.remove_object(bucket, object_name)
            logger.info(f"Deleted {bucket}/{object_name}")
            return True
        except S3Error as e:
            logger.error(f"Delete failed: {e}")
            return False

    async def list_files(self, bucket: str, prefix: str = "") -> list[str]:
        """列出文件"""
        self._ensure_bucket(bucket)
        objects = self.client.list_objects(bucket, prefix=prefix, recursive=True)
        return [obj.object_name for obj in objects]

# 全局实例
minio_client = MinioClient()

# 向后兼容的函数包装
async def upload_to_minio(file_path: str | Path, bucket: str, object_name: str) -> str:
    return await minio_client.upload_file(file_path, bucket, object_name)

async def download_from_minio(bucket: str, object_name: str, file_path: str | Path) -> Path:
    return await minio_client.download_file(bucket, object_name, file_path)
```

### 4. CORS 中间件未配置

**严重性**: 🟡 HIGH
**影响范围**: 前后端通信
**风险**: 生产环境前端无法调用后端 API

#### 当前问题

`backend/main.py` 中无 `CORSMiddleware` 配置，跨域请求会被浏览器阻止。

#### 修复方案

**文件**: `backend/main.py`
```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from backend.core.config import settings
from backend.core.metrics import prometheus_middleware
from backend.api.router import api_router

app = FastAPI(
    title="NextProject Backend v2",
    version="2.0.0",
    description="AI-driven website building platform"
)

# CORS 配置（在 prometheus_middleware 之前）
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:5173",      # Vite 开发服务器
        "http://localhost:3000",       # 前端生产环境
        "http://localhost:3002",       # 前端备用端口
        settings.frontend_url,         # 配置的前端 URL
    ],
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE", "PATCH", "OPTIONS"],
    allow_headers=["*"],
    expose_headers=["Content-Disposition"],  # 文件下载
)

# Prometheus 监控中间件
app.middleware("http")(prometheus_middleware)

# 路由
app.include_router(api_router)

@app.get("/health")
async def health_check():
    """健康检查端点"""
    return {"status": "healthy"}
```

```bash
#!/bin/bash
set -e

# 配置
MODULE_NAME="nextproject-main"

echo "========== Step 2: Push to Harbor =========="
docker push ${IMAGE_NAME}
docker tag ${IMAGE_NAME} ${IMAGE_FULL_NAME}

echo "========== Step 3: Login Apollo =========="

  -H "Content-Type: application/json" \
  -d '{
    "systemId": "'${SYSTEM_ID}'",
    "userAccount": "'${APOLLO_USER}'",
    "userPassword": "'${APOLLO_PASSWORD_ENCRYPTED}'"
  }' | jq -r '.data.token')

if [ -z "$TOKEN" ] || [ "$TOKEN" == "null" ]; then
  echo "ERROR: Login failed"
  exit 1
fi

echo "========== Step 4: Trigger Deployment =========="
RESULT=$(curl -s -X POST "${APOLLO_API}/devops/cicd/v1.0/job/deployByImage" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "systemId": "'${SYSTEM_ID}'",
    "appId": '${APP_ID}',
    "imageUrl": "'${IMAGE_FULL_NAME}'"
  }')

echo "Deployment Result: ${RESULT}"

JOB_ID=$(echo ${RESULT} | jq -r '.data.jobId')
if [ -z "$JOB_ID" ] || [ "$JOB_ID" == "null" ]; then
  echo "ERROR: Deployment failed"
  exit 1
fi

echo "========== Deployment Success =========="

echo "Image: ${IMAGE_FULL_NAME}"
```

### 8. 数据库连接池未优化

**严重性**: 🟡 MEDIUM
**影响范围**: 并发性能
**风险**: 高并发下连接耗尽

#### 修复方案

**文件**: `backend/core/database.py`
```python
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker
from backend.core.config import settings

# 优化连接池配置
engine = create_async_engine(
    settings.database_url,
    echo=settings.debug,
    pool_size=20,           # 常驻连接数
    max_overflow=40,        # 峰值额外连接数
    pool_timeout=30,        # 获取连接超时（秒）
    pool_recycle=3600,      # 连接回收时间（1 小时）
    pool_pre_ping=True,     # 连接前检查是否有效
)

AsyncSessionLocal = sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False,
    autocommit=False,
    autoflush=False
)

async def get_async_session() -> AsyncSession:
    async with AsyncSessionLocal() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise
        finally:
            await session.close()
```

```python
@router.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    await websocket.accept()
    client_id = str(uuid.uuid4())
    await manager.connect(client_id, websocket)

    # 心跳任务
    async def heartbeat():
        while True:
            try:
                await websocket.send_json({"type": "ping"})
                await asyncio.sleep(30)
            except Exception:
                break

    heartbeat_task = asyncio.create_task(heartbeat())

    try:
        while True:
            data = await websocket.receive_json()
            if data.get("type") == "pong":
                continue
            # 处理其他消息
    except WebSocketDisconnect:
        await manager.disconnect(client_id)
        heartbeat_task.cancel()
```
