## 三、生产环境（K8s + Apollo）

### 3.1 Harbor 镜像仓库

#### 配置信息

| 项目 | 值 |
|------|-----|
| **内网地址** | 192.168.1.18 |
| **公网地址** | harbor.trscd.com.cn |
| **Namespace** | ocean-km |
| **协议** | HTTP (内网) / HTTPS (公网) |

#### 镜像命名规范

```
格式：{harbor_domain}/{namespace}/{module_name}:{tag}

示例：
- 内网推送：192.168.1.18/ocean-km/site-builder:t20250320.1430
- K8s 拉取：harbor.trscd.com.cn/ocean-km/site-builder:t20250320.1430

标签规范：
- t{YYYYMMDD.HHMM}：时间戳标签（推荐）
- latest：最新版（不推荐生产使用）
- v{version}：语义化版本（如 v2.0.0）
```

#### 镜像推送流程

```bash
#!/bin/bash
# 构建并推送到 Harbor

MODULE_NAME="site-builder"
HARBOR_INTERNAL="192.168.1.18"
HARBOR_PUBLIC="harbor.trscd.com.cn"
NAMESPACE="ocean-km"

# 1. 生成时间戳标签
TIMESTAMP=$(date +%Y%m%d.%H%M)
TAG="t${TIMESTAMP}"
IMAGE_NAME="${MODULE_NAME}:${TAG}"

# 2. 构建镜像
echo "构建镜像：${HARBOR_INTERNAL}/${NAMESPACE}/${IMAGE_NAME}"
docker build --force-rm \
  -t "${HARBOR_INTERNAL}/${NAMESPACE}/${IMAGE_NAME}" \
  -f Dockerfile .

# 3. 推送到内网 Harbor
echo "推送镜像到内网 Harbor..."
docker push "${HARBOR_INTERNAL}/${NAMESPACE}/${IMAGE_NAME}"

# 4. 清理本地镜像（节省空间）
docker rmi "${HARBOR_INTERNAL}/${NAMESPACE}/${IMAGE_NAME}"

echo "✅ 镜像推送成功：${HARBOR_PUBLIC}/${NAMESPACE}/${IMAGE_NAME}"
echo "📦 可用于 K8s 部署"
```

#### 完整部署脚本（基于 ocean-km/update.sh）

```bash
#!/bin/bash
# deploy_to_apollo.sh - NextProject Apollo 部署脚本

set -e

# ==================== 配置区 ====================
MODULE_NAME="${1:-site-builder}"  # 模块名称（默认 site-builder）
HARBOR_INTERNAL="192.168.1.18"
HARBOR_PUBLIC="harbor.trscd.com.cn"
HARBOR_NAMESPACE="ocean-km"

APOLLO_API_BASE="http://192.168.1.15:90"
APOLLO_SYSTEM_ID="B6EF6BD39FE8C3DCA1B5E13AAD516BC3"

# 应用 ID 映射
declare -A APP_ID_MAP
APP_ID_MAP["km-app"]="53"
APP_ID_MAP["km-search"]="55"
APP_ID_MAP["site-builder"]="TBD"  # 待分配

APP_ID="${APP_ID_MAP[$MODULE_NAME]}"

if [ -z "$APP_ID" ] || [ "$APP_ID" == "TBD" ]; then
    echo "❌ 错误：应用 ${MODULE_NAME} 的 App ID 未配置，请联系 Apollo 管理员分配"
    exit 1
fi

# ==================== 1. 构建镜像 ====================
echo "============================================"
echo "📦 步骤 1/4: 构建 Docker 镜像"
echo "============================================"

TIMESTAMP=$(date +%Y%m%d.%H%M)
TAG="t${TIMESTAMP}"
IMAGE_NAME="${MODULE_NAME}:${TAG}"
FULL_IMAGE_INTERNAL="${HARBOR_INTERNAL}/${HARBOR_NAMESPACE}/${IMAGE_NAME}"
FULL_IMAGE_PUBLIC="${HARBOR_PUBLIC}/${HARBOR_NAMESPACE}/${IMAGE_NAME}"

echo "模块名称: ${MODULE_NAME}"
echo "镜像标签: ${TAG}"
echo "完整镜像: ${FULL_IMAGE_PUBLIC}"

# 检查 Dockerfile 是否存在
if [ ! -f "Dockerfile" ]; then
    echo "❌ 错误：未找到 Dockerfile"
    exit 1
fi

# 构建镜像
docker build --force-rm \
  -t "${FULL_IMAGE_INTERNAL}" \
  -f Dockerfile \
  .

if [ $? -ne 0 ]; then
    echo "❌ 镜像构建失败！"
    exit 1
fi

echo "✅ 镜像构建成功"

# ==================== 2. 推送镜像 ====================
echo ""
echo "============================================"
echo "📤 步骤 2/4: 推送镜像到 Harbor"
echo "============================================"

docker push "${FULL_IMAGE_INTERNAL}"

if [ $? -ne 0 ]; then
    echo "❌ 镜像推送失败！"
    exit 1
fi

echo "✅ 镜像推送成功"

# 清理本地镜像
sleep 3
docker rmi "${FULL_IMAGE_INTERNAL}"
echo "🗑️  本地镜像已清理"

# ==================== 3. Apollo 登录 ====================
echo ""
echo "============================================"
echo "🔐 步骤 3/4: 登录 Apollo 系统"
echo "============================================"

# 注意：这里的登录凭据是 RSA 加密的
# 实际使用时需要替换为真实的加密凭据
LOGIN_PAYLOAD='{
  "tel":"G3ZTp9bJygF/6nrTYc+JRSUCNn0GGjfpq92ULDKi0rGRuC4282Iyy8detP5wAPY56eCCCxE7KVExsJd+1fP0ASQjlHaJhfwgr0Aa2Q8cE5kYLyKAFWu3cMD9GkQaDAjltmW8ODT2JrdzmJ3+cNiZT2WbwddV+J5MUvCNAXirwa0=",
  "password":"g0USWa0498nV3Z3JRuQ2fj58DpOwooIepYYfmjhfz29P8JbAI96lwKLGCjIey6X83SiRjthzCqFVao9dGsx9QqH2q84FSC3M53zy5QcNuGE+5mj/PURZFrRJL5BC/gV0Kb6x2kfn3py2yNJze9A77sdTcUDL5cE3S9zK9jc1LDc=",
  "rememberMe":true,
  "random":49356
}'

LOGIN_RESPONSE=$(curl -s "${APOLLO_API_BASE}/apollo/user/login" \
  -H 'Accept: application/json, text/plain, */*' \
  -H 'Content-Type: application/json;charset=UTF-8' \
  -H "System-Id: ${APOLLO_SYSTEM_ID}" \
  -H 'Origin: http://192.168.1.15:90' \
  -H 'Referer: http://192.168.1.15:90/apollo-web/' \
  --data-raw "${LOGIN_PAYLOAD}" \
  --insecure)

if [ $? -ne 0 ]; then
    echo "❌ 登录请求失败！"
    exit 1
fi

# 解析 Token（支持两种格式）
TOKEN=$(echo "$LOGIN_RESPONSE" | grep -o '"token":"[^"]*"' | sed 's/"token":"\([^"]*\)"/\1/')

if [ -z "$TOKEN" ]; then
    echo "❌ 获取 Token 失败！"
    echo "登录响应：${LOGIN_RESPONSE}"
    exit 1
fi

echo "✅ 登录成功，Token: ${TOKEN:0:20}..."

# ==================== 4. 调用部署接口 ====================
echo ""
echo "============================================"
echo "🚀 步骤 4/4: 部署到 Kubernetes"
echo "============================================"

RANDOM_NUM=$((RANDOM % 100000))

DEPLOY_PAYLOAD=$(cat <<EOF
{
  "appId": ${APP_ID},
  "image": "${FULL_IMAGE_PUBLIC}",
  "random": ${RANDOM_NUM}
}
EOF
)

echo "部署参数："
echo "  - App ID: ${APP_ID}"
echo "  - 镜像: ${FULL_IMAGE_PUBLIC}"
echo "  - Random: ${RANDOM_NUM}"

DEPLOY_RESPONSE=$(curl -s "${APOLLO_API_BASE}/devops/cicd/v1.0/job/deployByImage" \
  -H 'Accept: application/json, text/plain, */*' \
  -H 'Content-Type: application/json;charset=UTF-8' \
  -H "System-Id: ${APOLLO_SYSTEM_ID}" \
  -H "X-User-Token: ${TOKEN}" \
  -H 'Origin: http://192.168.1.15:90' \
  -H 'Referer: http://192.168.1.15:90/apollo-web/' \
  --data-raw "${DEPLOY_PAYLOAD}" \
  --insecure)

if [ $? -ne 0 ]; then
    echo "❌ 部署请求失败！"
    exit 1
fi

echo "部署响应：${DEPLOY_RESPONSE}"

# 检查部署结果
if echo "$DEPLOY_RESPONSE" | grep -q '"code":"00000"'; then
    echo ""
    echo "============================================"
    echo "🎉 部署成功！"
    echo "============================================"
    echo "镜像：${FULL_IMAGE_PUBLIC}"
    echo "应用：${MODULE_NAME} (App ID: ${APP_ID})"
    echo "时间：$(date '+%Y-%m-%d %H:%M:%S')"
    echo "============================================"
else
    echo ""
    echo "============================================"
    echo "❌ 部署失败！"
    echo "============================================"
    echo "响应内容：${DEPLOY_RESPONSE}"
    exit 1
fi
```

## 五、故障排查

### 5.1 常见问题

**PostgreSQL 无法启动**
```bash
# 检查日志
docker compose logs postgres

# 常见原因：
# 1. 数据目录权限问题
sudo chown -R 999:999 ./data/postgres

# 2. 端口冲突
lsof -i :5432
```

**Redis 连接失败**
```bash
# 测试连接
docker compose exec redis redis-cli -a redis2025 ping

# 查看连接数
docker compose exec redis redis-cli -a redis2025 info clients
```

**Celery Worker 不消费任务**
```bash
# 检查队列状态
docker compose exec redis redis-cli -a redis2025
> LLEN celery

# 查看 Worker 日志
docker compose logs -f celery-worker

# 重启 Worker
docker compose restart celery-worker
```
