## 🔄 2026-04 本轮补充记录

### 代码与交互调整
- ✅ 新增旧编辑器路由兼容：访问 `/site-editor/:id` 时自动重定向到 `/sites/:id/edit`
- ✅ 站点编辑页补充失败可见性：
  - 路由参数为空时，直接展示「站点参数无效」错误态
  - 站点详情接口失败时，展示明确错误信息、当前路径、站点 ID 与返回列表入口
  - 在站点未成功加载前，禁用任务提交、重启、需求文档与历史任务读取
- ✅ 任务创建与重启失败时，编辑器内日志优先展示后端返回的 detail/message，避免只看到笼统失败文案

### 本轮验证结论
- ✅ `npm run build` 通过，验证前端路由修改与编辑器模板改动可完成生产构建
- ✅ 聚焦检查确认旧链接兼容与错误态逻辑已落到现有页面，不涉及额外路由守卫改造

### 新发现的一致性问题
- ⚠️ 认证跳转仍不一致：Axios 401 拦截器目前先 `router.replace('/')`，随后依赖路由守卫再跳登录页；与文档示例中的直接跳 `/login` 不一致
- ⚠️ 失败提示方式不一致：站点编辑页已使用页面内错误态，但全局请求层仍大量使用 `alert`，用户感知不统一
- ⚠️ 文档中的前端路由示例仍只描述 `/sites/:id/edit`，若对外保留旧入口，后续应在实施文档中标注「兼容旧链接」策略
