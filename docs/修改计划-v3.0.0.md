#### 新发现 UX / 鉴权 / 路由问题

| 类别 | 发现 | 影响 | 改进建议 |
|------|------|------|------|
| 路由口径 | 旧编辑页地址已变为兼容重定向，但文档和测试脚本仍可能沿用旧写法 | 用户、测试、文档口径不一致 | 统一正式入口为 **`/sites/:id/edit`**，旧地址只保留在兼容说明 |
| 登录前置 | 编辑页属于受保护路由，未登录会先进入 `/login?redirect=...` | 用户容易误以为编辑页可匿名访问 | 在 README、测试说明、页面提示中明确「编辑页需先登录」 |
| 预览 vs 管理权限 | `/preview/${siteId}` 可见不等于拥有 `/api/v2/sites/*` 的访问权限 | 容易出现「预览正常但接口 403」的割裂体验 | 在预览区、任务区、需求区增加权限来源和 403 说明 |
| 组织隔离可见性 | 组织成员关系和默认组织上下文对接口结果有直接影响，但前端提示不足 | 用户难以判断是资源不存在还是无权限 | 在编辑上下文中展示当前组织、角色来源，并为 403 提供明确解释 |
| 验证链路稳健性 | 主仓库格式/转义异常会直接阻断测试与解析 | 验证容易被无关格式问题打断 | 将依赖/代码格式清理纳入验证前检查，避免再次阻塞 |

#### 本轮建议追加到 v3.0.0 的体验改进项

1. **统一正式编辑页入口**：所有文档、导航、测试入口统一使用 **`/sites/:id/edit`**。
2. **补强登录前置提示**：让用户在进入编辑链路前就知道该页面受登录保护。
3. **补强 403 可解释性**：清晰区分「可预览」与「可管理」，避免把组织隔离误判为系统异常。
4. **展示组织上下文**：在站点编辑相关区域显示当前组织和权限来源，降低排障成本。
5. **加一层验证前清理检查**：先排除异常转义/依赖格式问题，再跑测试与容器验证。
